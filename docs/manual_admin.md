# 管理者マニュアル

STEKWIRED 運用担当者向け。管理画面の日常的な操作手順を説明する。

---

## 1. 管理画面へのアクセス

### URL

```
https://chat.stekwired.jp/admin/login.php
```

ローカル開発環境の場合:

```
http://localhost/ai-chat/admin/login.php
```

### 初期ログイン情報

| 項目 | 値 |
|---|---|
| メールアドレス | admin@stekwired.jp |
| パスワード | admin123 |

> **本番環境では初回ログイン後すぐにパスワードを変更すること（→ [5. パスワード変更](#5-ユーザーアカウントの管理)）**

### セッションについて

無操作状態が 1 時間続くと自動的にログアウトされる。ログアウトされた場合はメッセージが表示されるので再ログインする。

ログイン失敗を 5 回繰り返すとアカウントが 30 分ロックされる。ロック中はパスワードが正しくても「30 分ロック」メッセージが表示される。

## 2. クライアントの追加手順

新しいクライアント（導入企業）を登録し、埋め込みタグを発行する手順。

### 手順

1. 管理画面にログイン
2. 左メニューの「クライアント」をクリック
3. ページ上部の「新規追加」フォームに以下を入力

   | 項目 | 必須 | 入力内容 |
   |---|---|---|
   | 会社名 | ✅ | クライアントの正式な会社名 |
   | システムプロンプト | — | ボットの口調・役割・制約（後から変更可） |
   | お問い合わせ URL | — | チャットで解決しない場合の誘導先 URL |

4. 「追加する」をクリック
5. 一覧に追加され、`widget_key`（埋め込みキー）が自動発行される

### 埋め込みタグの発行

クライアント一覧の「埋め込みタグ」列に表示されているタグをクライアントに共有する。

```html
<script src="https://chat.stekwired.jp/widget.js"
  data-key="（発行されたキー）"></script>
```

このタグをクライアントサイトの `</body>` 直前に貼り付けるだけで、チャットボタンが表示される。

## 3. クライアント情報の変更

### 手順

1. 「クライアント」ページを開く
2. 変更したいクライアントの行の「編集」ボタンをクリック
3. インラインフォームが展開されるので内容を変更
4. 「保存」をクリック

### 変更可能な項目

| 項目 | 説明 |
|---|---|
| 会社名 | クライアント名 |
| システムプロンプト | ボットの口調・制約・役割を指定するプロンプト |
| お問い合わせ URL | フォールバック時のリンク先 |

> **`widget_key` は変更不可。** 変更するとクライアントサイトのチャットが動作しなくなる。

### システムプロンプトの書き方

システムプロンプトは Claude API に渡される指示文。以下のような内容を記載すると効果的。

```
あなたは株式会社○○のサポートアシスタントです。
丁寧で親しみやすい口調でお客様の質問に回答してください。
価格・在庫の詳細は「担当者にお問い合わせください」と案内してください。
```

## 4. FAQ の管理

### 4-1. 手動でのFAQ追加

1. 「FAQ」ページを開く
2. 「新規追加」フォームに入力

   | 項目 | 必須 | 説明 |
   |---|---|---|
   | クライアント | ✅ | 所属クライアントを選択 |
   | カテゴリ | — | 分類名（例: 料金・営業時間） |
   | 質問 | ✅ | よくある質問の文章 |
   | 回答 | ✅ | 質問に対する回答 |
   | キーワード | — | 空欄にすると Claude API が自動生成 |
   | 優先度 | — | 数値が高いほど検索上位に表示（デフォルト: 0） |

3. 「追加する」をクリック

### 4-2. CSV での一括インポート

多数の FAQ を一度に登録する際に使用する。

**CSV テンプレートのダウンロード**

FAQ 管理ページ上部の「サンプル CSV をダウンロード」リンクから取得する。

**CSV のフォーマット**

1 行目はヘッダー行。日本語・英語のどちらでも使用できる。

| 列名 | 必須 | 説明 |
|---|---|---|
| `カテゴリ`（または `category`） | — | FAQ のカテゴリ |
| `質問`（または `question`） | ✅ | 質問文 |
| `回答`（または `answer`） | ✅ | 回答文 |
| `キーワード`（または `keywords`） | — | カンマ区切り（空欄で自動生成可） |
| `優先度`（または `priority`） | — | 数値（デフォルト: 0） |

**注意事項**

- 文字コードは UTF-8（BOM 付きも可）
- `question` または `answer` が空の行はスキップされる
- 最大ファイルサイズは PHP の `upload_max_filesize` 設定に依存（通常 2MB 以上）

**インポート手順**

1. 「CSV インポート」セクションでクライアントを選択
2. 「キーワード自動生成」チェックボックスをオンにするとキーワード列が空の行を Claude API が補完（API コスト増に注意）
3. CSV ファイルを選択して「インポート実行」をクリック
4. 結果（成功件数・スキップ件数・エラー内容）が表示される

### 4-3. FAQ の編集

1. FAQ 一覧の「編集」ボタンをクリック
2. インラインフォームで内容を変更
3. 「保存」をクリック

キーワードを空欄にした状態で保存すると Claude API が自動生成する。

### 4-4. 公開/非公開の切り替え

各 FAQ 行の「公開中」/「非公開」ボタンをクリックすると即座に切り替わる。

- **公開中**: チャットの検索対象になる
- **非公開**: チャットの検索対象から除外される（回答に使われない）

### 4-5. FAQ の削除

各行の「削除」ボタンをクリックし、確認ダイアログで「OK」を選択する。削除したデータは元に戻せない。

### 4-6. キーワードの自動生成と手動修正

キーワードは FAQ 検索の精度に直結する。自動生成で作られたキーワードが不十分な場合は手動で編集する。

**キーワードの書き方**:
- カンマ区切りで複数入力（例: `料金,月額,費用,価格`）
- 同義語・表記ゆれを含めると検索精度が上がる
- スペースや記号は不要

### 4-7. 優先度の設定

優先度（`priority`）は FAQ 検索でスコアに加算される数値。デフォルトは 0。

- 同じキーワードに複数の FAQ がヒットした場合、優先度が高いものが優先表示される
- 重要度の高い FAQ（例: 料金・営業時間など頻出質問）に高い優先度を設定することを推奨

## 5. ユーザーアカウントの管理

### 5-1. 編集者アカウントの追加

クライアントの担当者に自社 FAQ を管理させる場合に編集者アカウントを作成する。

1. 「ユーザー」ページを開く
2. 「新規追加」フォームに入力

   | 項目 | 必須 | 説明 |
   |---|---|---|
   | メールアドレス | ✅ | ログイン ID（重複不可） |
   | パスワード | ✅ | 初期パスワード（担当者に別途通知） |
   | ロール | ✅ | `editor`（クライアント担当者）を選択 |
   | 所属クライアント | ✅ | 担当するクライアントを選択 |

3. 「追加する」をクリック

### 5-2. アカウントの削除

ユーザー一覧の「削除」ボタンをクリックする。自分自身のアカウントは誤操作防止のため削除できない。

### 5-3. パスワード変更

ヘッダー右上の「アカウント」リンクからパスワード変更ページにアクセスできる。

1. 現在のパスワードを入力
2. 新しいパスワードを 2 回入力
3. 「変更する」をクリック

> 編集者アカウントのパスワードを変更したい場合は、編集者本人がログイン後に「アカウント」から変更する。管理者側からは変更できない。

---

## 6. 会話ログの確認

1. 「会話ログ」ページを開く
2. ページ上部のプルダウンでクライアントを選択してフィルタ（「全クライアント」でも表示可）
3. 1 ページあたり 30 件表示・ページネーションで切り替え

**表示項目**: 日時 / クライアント名 / ユーザー入力 / ボット回答（80 文字抜粋）

チャットの傾向を把握して FAQ に反映させると回答精度が向上する。

---

## 7. トラブルシューティング

### チャットが応答しない

1. **API キーの確認** — `config/config.php` の `ANTHROPIC_API_KEY` が正しいか確認する
2. **PHP エラーログの確認** — XAMPP なら `C:\xampp\apache\logs\error.log` を確認する
3. **レート制限の確認** — 同一 IP からの短時間の大量アクセスで制限がかかっている場合がある（1 時間待てば解除）

### FAQ がヒットしない

1. **FAQ が公開状態か確認** — FAQ 一覧で「非公開」になっていないか確認
2. **キーワードを確認** — キーワード列が空、またはユーザーの質問と一致するキーワードがない
3. **スコアを確認** — 他の FAQ のスコアが高く、上位 3 件に入っていない可能性がある
4. **FAQ を追加・編集** — 実際のユーザー質問の言葉をキーワードに追加する

### ログインできない（ロック解除方法）

アカウントロック（30 分経過で自動解除）を手動で解除したい場合は、phpMyAdmin で以下を実行する：

```sql
UPDATE admin_users
SET login_failed_count = 0, login_locked_until = NULL
WHERE email = '解除したいメールアドレス';
```
