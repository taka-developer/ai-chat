# 問い合わせチャットボット システム仕様書

株式会社ステックワイアード | 2026年5月

---

## 1. 概要

STEKWIREDがクライアントサイトに提供する問い合わせチャットボット。Claude API（Anthropic）を使用して自作する。

### システム方針

- クライアントサイトにJavaScript 1行で埋め込み可能なウィジェット形式
- FAQデータをキーワード検索で絞り込み、Claude APIで自然言語回答を生成
- PHP + MySQL + Vanilla JS で構成し、外部インフラへの依存を最小化
- 複数クライアントをSTEKWIREDが一元管理する SaaS型運用

### 処理フロー概要

```
ユーザー入力
    ↓
① Claude API：キーワード抽出（3〜5語）
    ↓
② PHP + MySQL：FAQキーワード検索・スコアリング
    ↓
③ Claude API：ヒットFAQを元に自然言語で回答生成
    ↓
チャットUIに回答表示
```

## 2. FAQデータ構造・管理仕様

### 2.1 データベース設計

#### faqs テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| id | INT AUTO_INCREMENT | 主キー |
| client_id | INT | クライアントID（外部キー） |

| updated_at | DATETIME | 更新日時 |

#### clients テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| id | INT AUTO_INCREMENT | 主キー |
| name | VARCHAR(200) | クライアント名 |
| system_prompt | TEXT | チャットボットの口調・ルール・制約 |

### 2.2 キーワードフィールドの運用

キーワードはFAQ登録時にClaude APIが自動生成する。担当者は手動での追記・修正が可能。

| 方式 | メリット | デメリット |
|---|---|---|
| AI自動生成（推奨） | 担当者負担ゼロ・表記ゆれに強い | FAQ登録時にAPI1回消費 |
| 手動入力 | シンプル・コスト不要 | 抜け漏れが出やすい |
| AI生成＋手動追記（最良） | 精度・柔軟性ともに高い | — |

### 2.3 管理画面の権限設計

| ロール | 想定ユーザー | 操作範囲 |
|---|---|---|
| 管理者 | STEKWIRED担当者 | 全クライアントのFAQ・設定・ログ・クライアント追加 |
| 編集者 | クライアント担当者 | 自社FAQの追加・編集・削除・公開管理のみ |

### 2.4 管理画面の機能一覧

#### 管理者画面（STEKWIRED）

- クライアント一覧・追加・設定変更

- 埋め込みタグの発行

- FAQ一覧・追加・編集・削除
- カテゴリ管理
- キーワードの確認・手動追記
- 公開 / 非公開の切り替え

## 3. キーワード抽出・検索ロジック仕様

### 3.1 ① キーワード抽出（Claude API）

ユーザーの入力文をClaude APIに送り、検索キーワードを3〜5語抽出する。

**抽出プロンプト例**

```
以下のユーザー入力から検索キーワードを3〜5個抽出してください。
JSON配列のみで返してください。他の文字列は含めないこと。

入力：「土日って診てもらえますか？」
出力：["土日", "診療", "休日", "診察"]
```

キーワード抽出結果が空の場合はフォールバック処理へ移行する。

### 3.2 ② FAQ検索・スコアリング（PHP + MySQL）

抽出キーワードで `faqs` テーブルの `keywords` カラムと `question` カラムをLIKE検索し、ヒット数でスコアリングする。

**スコアリングロジック**

```
各FAQに対して：
  キーワードが keywords カラムに含まれる → +1点
  キーワードが question カラムに含まれる → +1点
  priority カラムの値を加算

スコア上位3件を取得してClaudeに渡す
```

**取得件数ルール**

| ヒット状況 | Claudeに渡すFAQ数 |
|---|---|
| 3件以上ヒット | スコア上位3件 |
| 1〜2件ヒット | ヒット分のみ |
| 0件（ヒットなし） | フォールバック処理へ |

**同スコア時の優先順位**

1. `priority` カラムの値が高いFAQを優先
2. `priority` も同値の場合は `updated_at` が新しいものを優先

### 3.3 ③ 回答生成（Claude API）

ヒットしたFAQをコンテキストとしてClaudeに渡し、自然言語で回答を生成する。

- ストリーミングレスポンス（SSE）を使用
- システムプロンプトにはクライアント固有の口調・制約を含める
- FAQに記載のない内容については問い合わせフォームへ誘導するよう指示

## 4. チャットUI・UX仕様

### 4.1 埋め込み方式

iframeを採用。WordPressテーマのCSSが干渉するリスクを回避する。

```html
<script src="https://chat.stekwired.jp/widget.js"
  data-key="{widget_key}"></script>
```

### 4.2 UI要素

| 要素 | 仕様 |
|---|---|
| 起動ボタン | 右下固定（position: fixed）・丸型・色はクライアントごとにカスタマイズ可 |
| チャット画面サイズ | 幅360px × 高さ520px（スマートフォンは全画面表示） |
| ヘッダー | ボット名・クライアントロゴ・閉じるボタン |
| メッセージ表示 | 吹き出し形式（左：ボット / 右：ユーザー） |
| 入力欄 | テキストエリア / Enterで送信 / Shift+Enterで改行 |
| 回答表示 | ストリーミングで文字を順次表示 |
| ローディング | 入力中はドット3つのアニメーション表示 |

### 4.3 会話UX

**初期表示**

- ボット：「こんにちは！ご質問をどうぞ。」
- サジェストボタンを最大5件表示（クライアントが管理画面で設定）

**回答後のアクション**

- 「この回答は参考になりましたか？」を表示
- 👍 はい → 会話継続
- 👎 いいえ → 「お問い合わせフォームへ」ボタンを表示

### 4.4 クライアントごとのカスタマイズ項目

| 項目 | 内容 |
|---|---|
| ボット名 | 「AIアシスタント」「〇〇クリニック案内」など自由設定 |
| メインカラー | 起動ボタン・吹き出し色 |
| 起動ボタンテキスト | 「お問い合わせ」「ご質問はこちら」など |
| サジェストボタン | 最大5件まで設定可 |
| フォールバック誘導先URL | 問い合わせフォームのURL |
| 営業時間外メッセージ | 時間外は自動で別メッセージを表示（時間はサーバー側で判定） |

## 5. エッジケース・フォールバック仕様

### 5.1 検索・回答に関するケース

| ケース | 対応 | 実装場所 |
|---|---|---|
| FAQヒット0件 | フォームへの誘導メッセージ＋ボタン表示 | PHP |
| FAQが0件（未登録） | 入力欄を非活性化・準備中メッセージ表示 | PHP + JS |
| 回答の確信度が低い場合 | 回答末尾に「詳しくはお問い合わせください」を自動付加 | システムプロンプト |

### 5.2 ユーザー入力に関するケース

| ケース | 対応 | 実装場所 |
|---|---|---|
| 空送信 | 送信ボタンを非活性化（入力がある時のみ活性） | JS |
| 200文字超の入力 | 入力欄で警告表示・送信不可 | JS |
| 連続送信（スパム） | 送信後3秒間は送信ボタンを非活性化 | JS |
| 意味のない入力 | キーワード抽出結果が空 → FAQヒット0件と同じ処理 | PHP |
| 関係のない話題・雑談 | 「その内容にはお答えできません」→ フォーム誘導 | システムプロンプト |

### 5.3 システム・通信に関するケース

| ケース | 対応 | 実装場所 |
|---|---|---|
| Claude APIタイムアウト（15秒） | エラーメッセージ＋リトライボタン表示 | PHP + JS |
| Claude APIエラー（500系） | エラーログ記録＋STEKWIRED宛に通知送信 | PHP |
| クライアント側ネットワーク切断 | 送信失敗メッセージ・入力内容は入力欄に保持 | JS |

### 5.4 悪用・セキュリティに関するケース

| ケース | 対応 | 実装場所 |
|---|---|---|
| プロンプトインジェクション | システムプロンプトで指示の上書き禁止を明記＋入力値をサニタイズ（htmlspecialchars） | PHP＋プロンプト |
| 個人情報の入力 | 個人情報を求めない・入力された場合はフォーム誘導（プロンプト制御） | システムプロンプト |
| 大量アクセス | IPアドレスごとに30回/時のレート制限・超過時はSTEKWIREDに通知 | PHP |

## 6. コスト試算

### 6.1 API費用（Claude Sonnet 基準）

- Input：$3 / 1Mトークン
- Output：$15 / 1Mトークン

### 6.2 月間コスト目安（1クライアントあたり）

| 規模 | 月間会話数 | 推定API費用 |
|---|---|---|
| 小規模（コーポレートサイト） | 〜200件 | $1〜3（約150〜450円） |
| 中規模（医療・公共） | 〜1,000件 | $5〜15（約750〜2,200円） |
| 大規模 | 〜5,000件 | $25〜75（約3,700〜11,000円） |

※ 1会話あたり平均1,000トークン想定（キーワード抽出＋回答生成の合計）

### 6.3 ビジネスモデル

- 初期構築費：15〜30万円
- 月額保守費：3〜5万円
- 実際のAPI費用：数百〜数千円/月
- → 保守費のほぼ全額が粗利
