## 1. 事前準備

### テスト用クライアント・ユーザーの準備

1. 管理画面にログイン（`admin@stekwired.jp` / `admin123`）
2. 「クライアント管理」→「新規追加」でテスト用クライアントを登録

   | 項目 | 値 |

   | システムプロンプト | あなたはテスト株式会社のサポートアシスタントです。 |
   | お問い合わせ URL | https://example.com/contact |

## 5. widget.js テスト

テスト用 HTML ファイル（`<body>` に以下を貼り付けてブラウザで開く）:

```html
<script src="http://localhost/ai-chat/public/widget.js"
  data-key="<widget_key>"></script>
```

| # | テスト項目 | 操作手順 | 期待結果 | 結果 |
|---|---|---|---|---|
| W-01 | 埋め込みボタン表示 | テスト HTML をブラウザで開く | 画面右下に 💬 ボタンが表示される | |
| W-02 | チャット開閉 | 💬 ボタンをクリック | チャットウィンドウが開く。再クリックで閉じる（ボタンアイコンが ✕ ↔ 💬 に切り替わる） | |
| W-03 | 無効な widget_key | `data-key="invalid_key_xxx"` で設置 | エラーにならず iframe が表示される（チャット画面側でエラー処理） | |

## 6. 合否判定基準

- 全項目 ✅ であれば本番デプロイ可
- 1 件でも ❌ がある場合はバグ修正後に再検証し、再検証後に ✅ を記録する
- 本番環境では最低限 C-01〜C-03 / A-01 / W-01〜W-02 を再実施すること
